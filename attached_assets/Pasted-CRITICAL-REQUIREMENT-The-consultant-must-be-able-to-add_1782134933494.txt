CRITICAL REQUIREMENT

The consultant must be able to add, edit, remove, disable, and create new numerology rules without touching code.

The system must function as a Numerology CMS.

The consultant is not a programmer.

All logic must be configurable from the admin panel.

---

RULE ENGINE

Create a dynamic rule engine.

Rules must come from the database.

The application must never rely on hardcoded numerology interpretations.

---
